  What changed:

  - In trainer.py, I added:

      - _build_shell_rollout_state(...)
      - _autotune_transition_thresholds(...)
  - SequentialTrainer.train_task(...) now runs that autotuner at the beginning of each task, right after support selection.

  What it does:

  - Uses a small task-start rollout signal from a few batches.
  - Builds deterministic shell-activity proxies for the current task.
  - Sweeps candidate values for:
      - demotion_threshold
      - promotion_threshold
      - max_demotions_per_step
  - Scores each candidate config against target promotion/demotion rates and picks the best one.
  - Updates the live shell_demotion_transweave config for the current task.

  Important detail:

  - This is off by default, so it won’t change existing behavior unless you enable config.transition_autotune.enable = True.
